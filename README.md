Реализация базового пайплайна взаимодействия с локальной большой языковой моделью (LLM).  
Развертывание сервера Ollama, интеграция с легковесной моделью `Qwen2.5:0.5B` и программная генерация отчетов через REST API.

---

## Описание задачи

В рамках лабораторной работы (базовый уровень) реализован программный комплекс для автоматизированного тестирования локальной языковой модели:
1. Развернут сервер **Ollama** для хостинга модели на локальных вычислительных мощностях.
2. Использована модель **Qwen2.5:0.5B** — легковесная архитектура, оптимизированная для быстрого инференса на CPU/GPU с ограниченной памятью.
3. Разработан Python-скрипт (`inference.py`), который отправляет пакет из 10 профильных технических запросов через HTTP API.
4. Сформирован структурированный аналитический отчет в форматах `.csv` и `.md` (Markdown с поддержкой абзацев внутри таблиц).

---

## Стек технологий

* **LLM Engine:** Ollama (REST API, порт `11434`)
* **Модель:** Qwen2.5:0.5B (0.5 млрд параметров)
* **Язык:** Python 3.10+
* **Библиотеки:** `requests` (HTTP-коммуникация), `pandas` (структурирование данных и экспорт), `tabulate` (формирование .md файла)

---

## Установка и подготовка окружения

Для воспроизведения результатов необходимо установить сервер Ollama и зависимости Python.

### 1. Установка сервера Ollama
* **Для Linux/WSL/Google Colab:**
  ```bash
  curl -fsSL [https://ollama.com/install.sh](https://ollama.com/install.sh) | sh
  ```
* **Для macOS/Windows:** Скачайте установщик с официального сайта [Ollama](https://ollama.com/download).

### 2. Загрузка весов модели

Запустите сервер Ollama (если он не запущен как системная служба) и скачайте модель:

```bash
# Эта команда автоматически скачает веса Qwen2.5 (около 350 МБ) и запустит модель
ollama run qwen2.5:0.5b
```

### 3. Установка Python-зависимостей

Перейдите в директорию проекта и установите необходимые библиотеки:

```bash
pip install -r requirements.txt
# или вручную:
pip install requests pandas tabulate
```

---

## Запуск скрипта инференса

Скрипт `inference.py` автоматически подключается к http://localhost:11434, отправляет запросы, парсит JSON-ответы и форматирует результаты.

```bash
python inference.py
```

**Ожидаемое поведение:** В консоли будет отображаться статус отправки каждого из 10 запросов. По завершении работы скрипт сгенерирует два файла:

* `inference_report.csv` — сырые данные, удобные для программной обработки.
* `inference_report.md` — отформатированная Markdown-таблица для интеграции в документацию.

## Результаты работы

Тестирование показало, что модель Qwen2.5:0.5B, несмотря на малый объем параметров, успешно справляется с базовыми техническими концепциями (сети, архитектура, базы данных), хотя и может допускать неточности в деталях из-за усеченного контекста.

После выполнения скрипта в директории появится файл inference_report.md - отчёт инференса с таблицей запросов и ответов LLM.
