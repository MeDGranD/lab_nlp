| Назови основные отличия протокола TCP от UDP.                                                          | TCP (Transmission Control Protocol) и UDP (User Datagram Protocol) являются двумя протокотипами данных сообщений, используемых в сетевых системах при передаче информации. Вот некоторые из их основных различий:<br><br>1. Функциональные принципы:<br>   - TCP: обеспечивает оптимальное хранение и отправка данных с минимальными затратами на прокоммуникацию.<br>   - UDP: не ограничивает размер данных, обеспечивает быструю передачу информации между точками.<br><br>2. Маршрут данных:<br>   - TCP: использует маршрутизацию при передаче данных по сети. Базовая скорость:<br>   - TCP: обеспечивает максимальная скорость передачи данных в рамках ограничений на интерфейсах (например, 65536 бит/квант/код).<br>   - UDP: не имеет ограничений по скорости передачи данных.<br><br>4. Транспортное решение:<br>   - TCP: обеспечивает функционирование через протокол с использованием протоколов соединения.<br>   - UDP: без специальных протоколов, что позволяет использовать простую технологию сетевой инфраструктуры для передачи данных.<br><br>5. Реализация:<br>   - TCP: реализован в операционных системах и сетевых приложениях.<br>   - UDP: реализован в клиент-серверах и используется для быстрого доступа к Интернету.<br><br>6. Рекомендуемые протоколы:<br>   - TCP (для обеспечения оптимального хранения и отправки данных с минимальными затратами на прокоммуникацию).<br>   - UDP (для быстрой передачи информации между точками).<br><br>7. Использование:<br>   - TCP: используется для прокоммундаций, связанных с передачей информации.<br>   - UDP: используется для быстрого доступа к интернету и обеспечения обратной связи передачи информации.<br><br>8. Влияние на технологию:<br>   - TCP: улучшил методы коммуникации между устройствами и позволяет эффективно передавать данные.<br>   - UDP: не изменяет структуру данных и используется для оптимизации скорости передачи информации.<br><br>9. | Кратко объясни алгоритм поиска кратчайшего пути Дейкстры.                                              | Для поиска кратчайшего пути в односвязном графе (диктатура) мы используем алгоритм Дейкстры. Алгоритм работает следующим образом:<br><br>1. **Задание данных**: <br>   - Массив, который содержит значения для всех вершин графа (входные данные).<br>   - Массив, который содержит индексы для хранения текущего состояния в очереди (`prev`), и статус (значение `dis`), а также номер текущей вершины (индекс 0) для текущей вершины.<br>   - Массив, который содержит значения для всех соседних вершин графа (входные данные).<br>   - Вспомогательный массив `parent` для хранения индексов родителей в дереве.<br><br>2. **Задание пути**: <br>   - Индекс текущей вершины: 0.<br>   - Сложность выхода из функции: O(1) (всего 3 шага).<br><br>3. **Циклическая очередь и статус `dis`**: <br>   - Используем циклическую очередь для хранения состояний текущей вершины.<br>   - `dis[0]` — индекс текущей вершины, и значение `parent[0]` — порядковый номер родительского дерева.<br><br>4. **Проверка свободных ядер:**<br>   - Проверяем, является ли `j` свободным (i.e., `dis[j] > i`).<br>   - Если да, то переход на новую вершину с искомым назначением `c`.<br><br>5. **Проверка правильности пути**: <br>   - В циклической очереди ставим новую вершину с искомым назначением.<br>   - Вычисляем идентификатор родителя для текущей вершины `parent[i] = j`.<br><br>6. **Ответ на выход**: <br>   - Если `dis[j] > i`, то выход: значение `i` — это искомый отрезок.<br><br>7. **Обновление статуса и перехода вниз**:<br>   - Или текущая вершина, или новую вершину с искомым назначением.<br><br>8. **Проверка свободного пути**: <br>   - Проверяем, есть ли еще свободные ядра для текущей вершины.<br>   - Если да, то переход на новую вершину с искомым назначением `j`.<br><br>9. **Обновление статуса и перехода вниз**:<br>   - Или текущая вершина, или новую вершину с искомым назначением.<br><br>10. **Завершение**: <br>    - Если выход не найден, то циклическая очередь становится пустой, и переменная `parent[dis[j]]` меняется на значение `j`, что означает, что пути из данной вершины невозможно по указанному пути.<br><br>Таким образом, алгоритм Дейкстры использует структуру временного хранения для хранения всех соседних вершин графа и для определения идентификаторов родителей. Он выполняет два шага: проверка свободных ядер и поиска пути, чтобы вычислить минимальный идентичный отрезок путей из заданной вершины до конечного назначения.<br><br>Этот алгоритм эффективен для графов размером \( O(n) \), где \( n \) — количество вершин в графе.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
